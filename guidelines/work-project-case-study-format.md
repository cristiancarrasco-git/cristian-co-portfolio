# Work project case study format (UX + product)

Use one file per project (six total). Keep scannable: short bullets, screenshots, diagrams.

Adapt the **Site structure** subsection to fit the project archetype:

- **Type A · Progressive / slide storytelling** *(e.g. Casa Lica)* — full-screen or slide-like progression that **stages curiosity** toward contact.
- **Type B · Data + interaction tool** *(e.g. Pokey Battle Predictors)* — **user POV** + **engineering truth** side by side.

---

## 1. Project card

| Field | Fill in |
|-------|---------|
| **Client / project** | Official name |
| **Role** | Design, dev, data, hybrid |
| **Timeline** | e.g. Q1–Q2 2026 |
| **Team** | Humans + collaborators |
| **Stack** | Frontend, backend, datastore, infra |
| **Outcome** | 1 measurable or qualitative sentence |

**Thesis**  
One sentence: what shifted for visitors and stakeholders?

---

## 2. Client ask → Delivered

Two parallel lists clarify scope without burying UX.

### What stakeholders wanted

Bullets grounded in kickoff feedback.

### What shipped

| Bucket | Contents |
|--------|----------|
| Brand *(if relevant)* | Identity, typography, verbal tone |
| Product / Site | IA, visuals, accessibility |
| Technical | Build, integrations, databases, hosting |

---

## 3. Journey (mandatory backbone)

Tell the visitor story **top → bottom**.

For **slide / staged sites**, add a subsection **Screen flow** naming each beats’ job (emotion, proof, services, conversion).

Standard macro-stages useful for recruiting readers:

1. **Discover**
2. **Orient**
3. **Evaluate / immerse**
4. **Commit**
5. **Handoff**
6. **Deepen**

For **prediction / tools**, also include a **Scenario table**: *Goal → Interaction → Emotional outcome.*

---

## 4. UX & methods mapped to phases

| Stage | UX methods | Artefacts |
|-------|-------------|-----------|
| Framing | Stakeholder sync, benchmarking | Brief |
| Structure | IA, storyboards | Sitemap |
| Detailed design | Components, prototyping | Figma |
| Evaluation | Cognitive walk-through, heuristic pass | Issue log |
| Launch | QA, instrumentation | Event naming doc |

Skills to cite when truthful: synthesis, IA, content design, IxD, visuals, WCAG-aware UI, analytics, facilitation.

---

## 5. Technical narrative *(optional but strong for hybrids)*

- Data model sketch
- Build stack
- How UI reads/writes persistence
- Honest constraints

---

## 6. Decisions & trade-offs

Short bullets — choice, sacrifice, why.

---

## 7. Impact

Metrics or quotes; backlog for next sprint.

---

## 8. Media

Assets + captions anchored to UX problem solved.

---

## 9. Credits

Collaborators & acknowledgements.
